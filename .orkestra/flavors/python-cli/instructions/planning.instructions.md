# Step 1: Planning

## Goal
Analyze the user request and create a detailed implementation plan for the CLI tool.

## Instructions
1. **Analyze Requirements:**
   - Identify the core purpose of the CLI.
   - List the required commands, subcommands, and arguments.
   - Determine necessary external libraries (if any).

2. **Design Architecture:**
   - Define the module structure (separation of concerns between CLI and logic).
   - Outline the main classes/functions in `core.py`.
   - Define the command-line interface spec (usage examples).

3. **Output:**
   - Create a file `docs/plan.md` containing:
     - **User Stories:** What the user can do.
     - **CLI Specification:** Help text draft.
     - **Technical Design:** File structure and key function signatures.
     - **Dependencies:** List of packages for `requirements.txt`.

4. **Validation:**
   - Ensure the plan covers all user requirements.
   - Check if the CLI design follows standard conventions (POSIX-like flags).
