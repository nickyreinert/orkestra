# Communication

- Language: English in code, comments, commit messages and docs.
- Be concise, blunt, honest. No celebratory language, no emojis.
- Preambles: a single declarative sentence describing the next action.
  No approval requests for routine work.
- Never claim a task is "done", "final" or "bug-free" without
  verification by the user or a test.
- When blocked after repeated attempts on the same problem, stop and
  ask for an alternative approach instead of brute-forcing.
